同步 vs 异步迭代器可视化演示

同步迭代器

function* syncIterator(max) { let idx = 0; while(idx < max) { yield idx++; } } for (const x of syncIterator(5)) { console.log(x); // 立即输出 }

异步迭代器

async function* asyncIterator(max) { let idx = 0; while(idx < max) { // 每个Promise有随机延迟 yield new Promise(resolve => { const delay = Math.floor(Math.random() * 1000); setTimeout(() => resolve(idx++), delay); }); } } // for await...of 会等待每个Promise解决 for await (const x of asyncIterator(5)) { console.log(x); // 按顺序输出，但时间不同 }

为什么异步迭代器也按顺序输出？

异步迭代器虽然每个 yield 的 Promise 解决时间是随机的，但 for await...of 循环会等待当前 Promise 被解决后，才继续下一次迭代。

这意味着：

  1. 迭代器首先 yield 第一个 Promise（值为 0）
  2. 循环等待这个 Promise 解决，然后输出 0
  3. 然后迭代器 yield 第二个 Promise（值为 1）
  4. 循环等待这个 Promise 解决，然后输出 1
  5. 依此类推...

所以即使每个 Promise 的解决时间是随机的，输出顺序仍然是 0, 1, 2, 3, 4，因为循环会等待每个值被解决后再继续。
